Руководство по Версии 1.0 Системы

Введение

 

Задачам настоящего этапа разработки Системы были:

  1. Отработка интерфейса Системы, включающего главное окно Системы, дерево меню, элементы управления и навигации;
  2. Разработка отдельных модулей Системы, обеспечивающих возможность работы с базовыми элементами Системы, лежащими в основе всех остальных элементов Системы, а именно:
    • Модуля Администрирования, обеспечивающего ведение данных о Пользователях Системы, их правах доступа к данным, модулям Системы и элементам управления;
    • Модулей, обеспечивающих ведение Справочников и Словарей, необходимых для работы Системы;
    • Элементов Модуля «Карточки Клиентов», обеспечивающих работу с основными данными о Клиентах Компании (Название, адрес, основной представитель и представители и связь с ними, реквизиты Клиента, Субподрядчики и родственные Компании Клиента) – как основы для наращивания всей функциональности работы с Карточками Клиентов, а также, как заготовки для последующего создания модуля работы с Карточками Субподрядчиков;
    • Элементов модулей работы с Обращениями и Запросами Клиентов обеспечивающих учет в системе данных об Обращениях и Запросах Клиентов, как основы для наращивания всей функциональности по работе с Делами и Заявками Клиентов;
    • Элементов модуля работы с тарифами в Системе (пока только в части входящих тарифов тарифов «ХАБ – Зоны» как основы для использования тарифов в процессе подготовки тарифов Компании и КП Клиентам, а также, как заготовки для последующего создания на их основе модулей ведения данных о входящих тарифах типа «Хаб – Хаб» и модулей аедения данных о Тарифах Компании;
    • Технологии подготовки отчетов на основании данных, имеющихся в Системе (на примере заявки на подпись Клиенту).
  3. Согласование с Компанией форм представления данных о тарифах, Хабах и Зонах с целью разработки механизмов загрузки в Систему данных о входящих Тарифах;
  4. Наполнение базы данных Системы в части:
    • Основных Справочников и Словарей Системы;
    • Данными об отдельных пользователях Системы;
    • Данными об отдельных тарифах (в рамках процесса подготовки механизмов загрузки в Систему данных о тарифах).

 

Система находится в стадии разработки, поэтому многие модули работают не в полном объеме или только «обозначены» для лучшего понимания того, что и где будет находиться, а многие модули ветви даже и не обозначены.

Настоящее руководство по версии 1.0 Системы ни в коей мере не следует рассматривать в качестве полноценной документации на Систему не только из-за того, что Система находится в ранней стадии разработки, но и потому, что на подготовку данного Руководства Исполнителю было отведено  очень мало времени. Тем не менее, в результате проверки Заказчиком Системы в ее нынешнем состоянии хотелось бы получить рекомендации, прежде всего, по реализации основных элементов интерфейса Системы.

Главное окно Системы

В правой части – три окна, предназначенные для отображения данных следующих списков:

  • «Сообщения входящие» – это горизонтальные задачи от других пользователей Системы;
  • «Сообщения по регламенту» - сообщения о сработавших таймерах;
  • Сообщения (исходящие) – статус горизонтальных задач, выданных данным пользователем другим пользователям.

Эти же списки должны будут быть доступны пользователю и при работе с любым модулем системы. Для этого предусмотрены кнопки (1,2,3) которые расположены в правой части окна. При свершении того или иного события (срабатывание таймера, получение новой горизонтальной задачи и т.д.) соответствующая кнопка будет подсвечиваться и по нажатию на нее пользователю будет показано содержание соответствующего списка. В данной версии Системы эти возможности не реализованы.

Управление поиском данных

В окнах, обеспечивающих работу с табличным представлением данных предусмотрена возможность обеспечения поиска данных – отбора данных по заданным пользователем критериям. Для этого в таких окнах  предусмотрена группа полей, в которые пользователь может ввести значения критериев для поиска. Эта группа полей размещена в верхней части таких окон. Например:

При нажатии на кнопку «Поиск» происходит выборка дынных по выбранным критериям. При нажатии на кнопку «Сброс» значения выбранных критериев поиска очищаются.

 

Для экономии места должна иметься возможность скрыть панель поиска:

При нажатии на эту кнопку панель поиска скрывается и остается больше места для просмотра таблицы. При желании панель поиска можно вернуть, нажав на эту же кнопку.

Примечания:

1.Возможность скрытия панели поиска реализована, пока, не во всех формах данной Версии Системы. На те формы, где она не реализована она будет распространена. Кроме того, в некоторых формах, например в формах работы с тарифами, для скрытия панели поиска используется другая кнопка – в этих формах она будет заменена на стандартную .

2. В формах работы с тарифами возможности поиска данных, пока не реализованы или реализованы не до конца.
